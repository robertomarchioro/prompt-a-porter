Design System

Tokens, scale e tre componenti chiave come prova del concept. Tutto via CSS Custom Properties, senza framework. Dark e light side-by-side; tre temperature neutre selezionabili.

Tone
tokens.css · tokens.json

Razionale. Il prodotto è invocato decine di volte al giorno tramite hotkey — la tipografia di sistema garantisce zero costo di rendering all'apertura del Command Palette, mentre JetBrains Mono è preferito a Fira Code perché le ligature confondono i {{ dei segnaposti. Radii morbidi (6/10/14) e ombre definite seguono Raycast: il prodotto deve sembrare un tool, non un'app web.

Razionale neutri. Tre toni (slate, stone, zinc) per testare in contesto la temperatura giusta prima di committare. Croma sempre < 0.02 sui background → percepiti come "grigi" ma con un'identità calda/fredda riconoscibile a colpo d'occhio. Dark di default per utenza developer; il light non è un negativo del dark — è ricostruito con luminanze appropriate per WCAG AA (testo default ≥ 7:1 sul canvas).

Razionale Button. Altezza standard 32 px (sm 26, lg 40) — corrisponde alla densità "bilanciata" richiesta. Il .kbd integrato nel pulsante invece che a fianco è una scelta deliberata: in app keyboard-first lo shortcut è parte del bottone, non un'aggiunta. btn--primary usa l'accent team (viola) perché è l'azione contestuale di default in un mondo collaborativo; btn--private usa l'ambra solo quando l'azione cambia visibilità del prompt — mai come sostituto del primary.

Razionale highlighting. I segnaposti sono cittadini di prima classe: il principio è che chi guarda il body deve vedere immediatamente quali sono i parametri.
· A · pill: massima leggibilità, ma rischia di "frammentare" la lettura del prompt come prosa — meglio in editor con poco testo e tanti placeholder.
· B · underline: il prompt resta leggibile come testo, il segnaposto è una "parola speciale". Più rispettoso del flusso, meno aggressivo. La mia raccomandazione di default.
· C · bordo: middle ground — più tecnico, evoca un campo. Buono per il renderer dove i segnaposti diventano click-target.
In tutti i casi le graffe {{ }} sono al 55-70% di opacità: aiutano a parsare ma non rubano l'occhio.
